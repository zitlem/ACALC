    Implement the recursive descent parser following the grammar above. Key implementation details:

    1. Public `evaluate(input: String): Double?` method:
       - Trim and strip all whitespace from input
       - Return null for empty string
       - Wrap parsing in try/catch, return null on any exception
       - After parsing, check `result.isNaN() || result.isInfinite()` — return null if true
       - Verify all input was consumed (pos == input.length) — return null if not

    2. Private inner `Parser` class with `pos: Int` cursor:

    4. Do NOT use regex for tokenization — scan character-by-character for simplicity.
    5. The parser operates on Double — precision for expression entry is sufficient. Conversion precision is handled by ConversionEngine with BigDecimal.

```kotlin
package com.acalc.domain

class ExpressionEvaluator {

    fun evaluate(input: String): Double? {
        val cleaned = input.trim().replace("\\s+".toRegex(), "")
        if (cleaned.isEmpty()) return null
        return try {
            val parser = Parser(cleaned)
            val result = parser.parseExpression()
            // Verify all input consumed — leftover chars means malformed input
            if (parser.pos != cleaned.length) return null
            if (result.isNaN() || result.isInfinite()) null else result
        } catch (e: Exception) {
            null
        }
    }

    private class Parser(private val input: String) {
        var pos = 0

        // expression = term (('+' | '-') term)*
        fun parseExpression(): Double {
            var result = parseTerm()
            while (pos < input.length && (input[pos] == '+' || input[pos] == '-')) {
                val op = input[pos++]
                val right = parseTerm()
                result = if (op == '+') result + right else result - right
            }
            return result
        }

        // term = factor (('*' | '/') factor)*
        private fun parseTerm(): Double {
            var result = parseFactor()
            while (pos < input.length && (input[pos] == '*' || input[pos] == '/')) {
                val op = input[pos++]
                val right = parseFactor()
                if (op == '/' && right == 0.0) {
                    throw ArithmeticException("Division by zero")
                }
                result = if (op == '*') result * right else result / right
            }
            return result
        }

        // factor = '-' factor | '(' expression ')' | NUMBER
        private fun parseFactor(): Double {
            // Unary minus
            if (pos < input.length && input[pos] == '-') {
                pos++
                return -parseFactor()
            }
            // Parenthesized expression
            if (pos < input.length && input[pos] == '(') {
                pos++ // consume '('
                val result = parseExpression()
                if (pos < input.length && input[pos] == ')') {
                    pos++ // consume ')'
                } else {
                    throw IllegalArgumentException("Missing closing parenthesis")
                }
                return result
            }
            // Number
            return parseNumber()
        }

        private fun parseNumber(): Double {
            val start = pos
            while (pos < input.length && (input[pos].isDigit() || input[pos] == '.')) {
                pos++
            }
            if (pos == start) {
                throw IllegalArgumentException("Expected number at position $pos")
            }
            return input.substring(start, pos).toDouble()
        }
    }
}
```

Key design decisions:
- Division by zero throws ArithmeticException, caught by outer try/catch, returns null
- Unary minus is handled in parseFactor (allows `-3`, `5 + -2`, `-(3+4)`)
- `pos != cleaned.length` check catches trailing operators (e.g., `2 + `) because the parser stops at the invalid point
- `2 + + 3` fails because after consuming `2 +`, parseTerm calls parseFactor which calls parseNumber, and `+` is not a digit — throws, returns null
- `* 3` fails because parseExpression calls parseTerm calls parseFactor calls parseNumber, and `*` is not a digit — throws, returns null
- No external libraries used. No Android imports. Pure Kotlin.
