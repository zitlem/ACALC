## Component Inventory

### New: Swap Button

- **Component:** `IconButton`
- **Icon:** `Icons.Default.SwapVert`
  - Rationale: the rows are stacked vertically; SwapVert communicates a vertical
    reversal of the row order. SwapHoriz would imply left-right field swap, which
    is not what the button does.
- **Placement:** Between the last tab row and the first `UnitRowCard`, inside the
  `LazyColumn` as a dedicated `item {}` block positioned before `itemsIndexed(state.rows)`.
  Alternatively: as a `stickyHeader {}` above the rows if the list is long enough
  to scroll. Default: plain `item {}` immediately before row items.
- **Alignment:** Centered horizontally (`Modifier.fillMaxWidth()` + `wrapContentWidth(Alignment.CenterHorizontally)`).
- **Size:** Default `IconButton` (48.dp touch target). Icon size: 24.dp.
- **State:** Always enabled as long as `state.rows.size >= 2`. If for any reason
  only 1 row exists (guard), the button is `enabled = false`.
- **Content description:** `"Swap units"`
- **Callback signature added to `ConverterContent`:** `onSwapRows: () -> Unit`

### New: Copy Icon (per row)

- **Component:** `IconButton`
- **Icon:** `Icons.Default.ContentCopy`
- **Placement:** Trailing position inside `UnitRowCard`'s Row, after the unit dropdown.
  The current row layout is:
  `[OutlinedTextField] [Spacer 8dp] [UnitDropdown 88dp] [Close or Spacer 48dp]`
  Replace the trailing `Spacer(Modifier.width(48.dp))` (shown when `!canRemove`) and
  add a copy `IconButton` as an additional trailing element when `canRemove` is false.
  When `canRemove` is true, the row shows both Close and Copy icons.
  Layout becomes:
  `[OutlinedTextField weight(1f)] [Spacer 8dp] [UnitDropdown 88dp] [CopyIconButton] [CloseIconButton if canRemove]`
- **Enabled:** `row.value.isNotEmpty()`
- **Tint when enabled:** `MaterialTheme.colorScheme.onSurfaceVariant`
- **Content description:** `"Copy value"`
- **Callback signature added to `ConverterContent`:** `onCopyRow: (Int) -> Unit`

### Existing: Snackbar (wiring required)

`AppShell` currently uses `Scaffold` without a `SnackbarHost`. This phase must wire it.

- Add `val snackbarHostState = remember { SnackbarHostState() }` in `AppShell`.
- Pass `snackbarHost = { SnackbarHost(snackbarHostState) }` to `Scaffold`.
- Pass `snackbarHostState` down to `ConverterScreen` and then to `ConverterContent`
  as a parameter: `snackbarHostState: SnackbarHostState`.
- The copy action calls `snackbarHostState.showSnackbar(...)` inside a `LaunchedEffect`
  or a coroutine scope obtained via `rememberCoroutineScope()`.
- **Duration:** `SnackbarDuration.Short` (approximately 4 seconds on Android).
- **No action button** on the Snackbar (action = null). Copy is not undoable.

## Number Display Contract

This formalizes decisions D-10 and D-11 from CONTEXT.md as a precise implementation spec
for `formatConverted()` in `ConverterViewModel`.

### Rules (applied in order)

1. **Zero:** return `"0"`
2. **Scientific notation threshold â€” small:** if absolute value < `0.000001` (i.e., `< 1e-6`),
   format as scientific notation with up to 6 significant digits: e.g., `1.23457e-8`
3. **Scientific notation threshold â€” large:** if absolute value > `999_999_999` (i.e., `> 9.99999999e8`),
   format as scientific notation with up to 6 significant digits: e.g., `1.23457e+10`
4. **Normal range:** strip trailing zeros, cap at 8 decimal places using `RoundingMode.HALF_UP`
5. **Thousands separators:** do NOT add thousands separators to converter output fields.
   The value field content is editable by the user; inserting commas would corrupt
   re-entry. Display is the raw numeric string only.

### Kotlin implementation target

```
private fun formatConverted(value: BigDecimal): String {
    if (value.compareTo(BigDecimal.ZERO) == 0) return "0"
    val abs = value.abs()
    return when {
        abs < BigDecimal("0.000001") || abs > BigDecimal("999999999") ->
            "%.6e".format(value.toDouble())
        else ->
            value.setScale(8, RoundingMode.HALF_UP)
                 .stripTrailingZeros()
                 .toPlainString()
    }
}
```

The exact format string for scientific notation (`%.6e`) produces `1.234568e-08` on the
JVM. This is acceptable for a personal-use app. Executor may adjust to `%s` formatting
via `BigDecimal.toEngineeringString()` if preferred, but must match the threshold rules.

## Interaction Contracts

### Swap Rows (CONV-11)

| State | Behavior |
|-------|----------|
| Tap swap button (2 rows) | Rows list is reversed. Row 0 becomes Row 1 and vice versa. `activeRowIndex` is set to `null` (cleared). All non-active rows are recomputed from the new row configuration. |
| Tap swap button (3+ rows) | Full list reversal. First row becomes last. `activeRowIndex` cleared. |
| Row count < 2 | Button is `enabled = false`. Tap has no effect. |
| Animation | No custom animation required. Compose recomposition handles the list reorder. Smooth enough for a personal app; no `AnimatedContent` or `animateItemPlacement` required unless executor chooses to add it. |

### Copy to Clipboard (CONV-12)

| State | Behavior |
|-------|----------|
| Tap copy icon (non-empty value) | `ClipboardManager.setPrimaryClip(ClipData.newPlainText("", row.value))` using `LocalClipboardManager.current`. Show Snackbar "Copied" with `SnackbarDuration.Short`. |
| Tap copy icon (empty value) | Button is `enabled = false`. Not tappable. |
| Snackbar queuing | If user taps copy rapidly on multiple rows, each Snackbar replaces the previous one (`showSnackbar` cancels the current Snackbar before showing the new one â€” this is the default `SnackbarHostState` behavior). |

### APK Build (APP-03)

No UI contract. Build-only requirement: `./gradlew assembleDebug` produces a valid APK
at `app/build/outputs/apk/debug/app-debug.apk`. No permissions UI, no settings screen.
