    **Structure — two composables per D-11 and research Pattern 3:**

    1. `CalculatorScreen(modifier: Modifier)` — public, creates ViewModel via `viewModel { CalculatorViewModel() }`, collects state via `val state by viewModel.state.collectAsStateWithLifecycle()`, delegates to `CalculatorContent`.

    2. `CalculatorContent(state: CalculatorState, onDigit, onOperator, onDecimal, onClear, onBackspace, onPercent, onEquals, modifier)` — private, stateless, receives state + lambdas. This enables @Preview without a ViewModel.

    **CalculatorContent layout:**

    ```
    Column(modifier.fillMaxSize()) {
        // TOP: Display area (takes remaining space via weight(1f))
        DisplayArea(expression, result, isError)

        // BOTTOM: Button grid (fixed height, fills width)
        ButtonGrid(onDigit, onOperator, onDecimal, onClear, onBackspace, onPercent, onEquals)
    }
    ```

    **DisplayArea:**
    - `Column` with `Modifier.fillMaxWidth().padding(horizontal = 24.dp, vertical = 16.dp)`, content aligned to bottom-end
    - Expression line: `Text(state.expression)` with `MaterialTheme.typography.headlineMedium`, single line, `textAlign = TextAlign.End`, `maxLines = 1`, `overflow = TextOverflow.Ellipsis`, color = `MaterialTheme.colorScheme.onSurface`
    - Result line: `Text(state.result)` with `MaterialTheme.typography.displaySmall`, single line, `textAlign = TextAlign.End`, color = if `state.isError` then `MaterialTheme.colorScheme.error` else `MaterialTheme.colorScheme.onSurfaceVariant`
    - Use `Spacer(Modifier.weight(1f))` above the text lines to push them to the bottom

    **ButtonGrid — 5 rows x 4 columns per D-01:**
    Use `Column` of `Row`s. Each `Row` has `Modifier.fillMaxWidth()`. Each button uses `Modifier.weight(1f)` in RowScope to fill equally. Add `Modifier.padding(4.dp)` around each button for spacing. Buttons should have `Modifier.aspectRatio(1f)` for square shape (or close to it — adjust if it doesn't fit; can use fixed height instead, e.g., `Modifier.height(72.dp)`).

    Layout per D-01:
    ```
    Row 1: C       <-      %       /
    Row 2: 7       8       9       x
    Row 3: 4       5       6       -
    Row 4: 1       2       3       +
    Row 5: 0 (weight 2f)   .       =
    ```

    Row 5: `0` button gets `Modifier.weight(2f)` to span two columns. `.` and `=` get `Modifier.weight(1f)` each.

    **Button styling per D-03 — three visual categories using Material 3 components:**

    | Category | Keys | Composable | Why |
    |----------|------|------------|-----|
    | Digits | 0-9, . | `FilledTonalButton` | secondaryContainer background by default — subtle |
    | Operators | +, -, x, / | `Button` (filled) | primary color — stands out |
    | Actions | C, backspace, % | `OutlinedButton` | outlined style — distinct from both |
    | Equals | = | `Button` with `containerColor = MaterialTheme.colorScheme.tertiary` | Accent color for the primary action |

    Use `ButtonDefaults.buttonColors(containerColor = MaterialTheme.colorScheme.tertiary, contentColor = MaterialTheme.colorScheme.onTertiary)` for the equals button.

    **Button text:**
    - Digits: the digit character, `MaterialTheme.typography.titleLarge`
    - Operators: the symbol (+, -, x, /), same typography
    - Backspace: use text "⌫" (unicode U+232B) or `Icons.AutoMirrored.Filled.Backspace` if available — check material-icons-core. If icon not available, use text "<-" or "⌫".
    - C, %, =: text labels

    **Button click handlers — map each button to the correct ViewModel method:**
    - Digits 0-9: `onDigit("0")` through `onDigit("9")`
    - Operators: `onOperator("+")`, `onOperator("-")`, `onOperator("x")`, `onOperator("/")` — pass "x" not "*" per research (ViewModel handles substitution)
    - Decimal: `onDecimal()`
    - C: `onClear()`
    - Backspace: `onBackspace()`
    - %: `onPercent()`
    - =: `onEquals()`

    **Shape:** Use `RoundedCornerShape(16.dp)` for buttons via `shape` parameter for a modern feel.
