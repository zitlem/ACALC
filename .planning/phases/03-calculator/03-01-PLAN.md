    **State model** (per D-11):
    ```kotlin
    data class CalculatorState(
        val expression: String = "",
        val result: String = "",
        val isError: Boolean = false
    )
    ```
    Plus a private `resultShown: Boolean` field in the ViewModel (not in the state class — UI doesn't need it).

    **Internal expression field:** Private `var expression = ""` that tracks the raw expression string. Updated alongside `_state.value`.

    **onDigit(digit: String)** (per D-04):
    - If `resultShown` is true, clear expression and start fresh with the digit
    - Append digit to expression
    - Update state, set `resultShown = false`

    **onOperator(op: String)** (per D-07, pitfall 3 and 4):
    - If expression is empty: only allow "-" (unary minus), ignore all others
    - If last char is an operator: replace it with the new operator (don't append)
    - If `resultShown` is true: continue from current result value (strip formatting from result, use as base)
    - Otherwise: append operator
    - Update state, set `resultShown = false`

    **onDecimal()** (per D-05, pitfall 1):
    - Find current token: scan backwards from end of expression to find last operator (+, -, x, /)
    - If current token already contains ".": do nothing
    - If expression is empty or last char is an operator: prepend "0" before "." (e.g., "0.")
    - Append "."
    - Update state, set `resultShown = false`

    **onClear()** (per D-06):
    - Reset expression to ""
    - Set state to `CalculatorState()` (all defaults)

    **onBackspace()** (per D-06, pitfall 7):
    - If expression is empty: do nothing
    - Drop last character from expression
    - If result is just "-": clear to ""
    - Update state with new expression, clear result, set `resultShown = false`

    **onPercent()** (per CALC-04, research open question 2):
    - Find current token (same scan as onDecimal)
    - If current token is empty: do nothing
    - Replace current token with "(currentToken/100)" OR evaluate the full expression with current token having /100 appended
    - Simpler: just evaluate the entire `expression` with the current number token replaced by `(token/100)`, then format and show result
    - Actually simplest per research: treat % as "evaluate current expression with last token divided by 100". Append "/100" to expression, then call onEquals() logic.
    - Set `resultShown = true`

    **onEquals()** (per D-04, D-09, D-10, pitfall 2 and 5):
    - If expression is empty: do nothing
    - Sanitize: `expression.trimEnd { it in "+-x/" }.replace("x", "*")` (per D-12, x-to-* substitution)
    - Call `evaluator.evaluate(sanitized)`
    - If null: set result to "Error", isError to true
    - If valid Double: format with `formatResult(value)`, set result, isError to false
    - Store unformatted result (no commas/spaces) back into `expression` for continued operations
    - Set `resultShown = true`

    **formatResult(value: Double): String** (per D-08, D-10, pitfall 6):
    - If `value == floor(value)` and value is not infinite and fits in Long range: format as integer with `NumberFormat.getIntegerInstance().format(value.toLong())`
    - Otherwise: format with `NumberFormat.getNumberInstance()` with `maximumFractionDigits = 10` and `isGroupingUsed = true`
    - This handles CALC-07 (thousands separators) and D-10 (no trailing zeros)
