  <implementation>
    GREEN phase implementation:

    **State model (top of ConverterViewModel.kt, before the class):**

    - `sealed class UnitPair` with 6 subclasses: Length, Weight, Volume, Temperature, Area, Speed. Each holds `from` and `to` typed enum fields.
    - `data class CategoryState(val topInput: String = "", val bottomInput: String = "", val units: UnitPair)`
    - `enum class ActiveField { TOP, BOTTOM }`
    - `data class ConverterState(val selectedCategory: UnitCategory = UnitCategory.LENGTH, val activeField: ActiveField = ActiveField.TOP, val topInput: String = "", val bottomInput: String = "", val units: UnitPair = UnitPair.Length(LengthUnit.MM, LengthUnit.INCH))`

    **ConverterViewModel class:**
    - Extends `ViewModel()` (per D-11, same as CalculatorViewModel pattern)
    - Private `evaluator = ExpressionEvaluator()` and `engine = ConversionEngine()`
    - `_state = MutableStateFlow(ConverterState())` / `val state: StateFlow<ConverterState> = _state` (per D-11)
    - Private `categoryStateMap: MutableMap<UnitCategory, CategoryState>` initialized via companion object `defaultStateFor()` for all 6 categories (Pitfall 5)
    - `onTopChanged(raw: String)`: set activeField=TOP, update topInput=raw, evaluate with ExpressionEvaluator (per D-10), if null return without touching bottomInput (Pitfall 6), else convert via engine and set bottomInput=formatConverted(result) (per D-01, D-02, D-03)
    - `onBottomChanged(raw: String)`: mirror of onTopChanged for bottom field (per D-12 activeField guard)
    - `onCategorySelected(category: UnitCategory)`: save current CategoryState to map, load new category state from map, update _state (per D-07, CONV-10)
    - `onTopUnitChanged(...)` and `onBottomUnitChanged(...)`: update the UnitPair in state, then recompute using the active field's current input. Use a helper to create the new UnitPair matching the current category. These take the unit index or enum value and create the appropriate sealed UnitPair subclass.
    - `convertForCurrentCategory(value: BigDecimal, fromTop: Boolean): BigDecimal`: when-dispatch on sealed UnitPair to call correct engine.convert() overload (per D-10)
    - `formatConverted(value: BigDecimal): String`: setScale(10, HALF_UP).stripTrailingZeros().toPlainString()
    - Use `BigDecimal(evaluated.toString())` to avoid Double->BigDecimal precision trap (Pitfall 4)

    **Helper functions for UI (on ConverterState or as ViewModel methods):**
    - `getUnitsForCategory(category: UnitCategory): List<Pair<String, String>>` returning list of (enumName, displayName) for dropdowns
    - `getTopUnitDisplayName()` / `getBottomUnitDisplayName()` extracting from sealed UnitPair
